# Contributing to LAN9252 EtherCAT SPI Evaluation Board

Thank you for your interest in contributing to this project! We welcome all forms of contributions including bug reports, design improvements, documentation, and examples.

## How to Contribute

### 1. Report Bugs
- Use the GitHub Issues tab
- Provide a clear description of the issue
- Include photos/diagrams if hardware-related
- Mention your setup details (tools, versions, environment)

### 2. Suggest Improvements
- Design optimizations
- Manufacturing recommendations
- EMI/Signal integrity improvements
- Cost reduction opportunities
- Component alternatives

### 3. Contribute Documentation
- Better guides for fabrication
- Firmware examples
- Integration tutorials
- Troubleshooting guides

### 4. Submit Enhancements
- Design revisions
- Additional layers or variants
- Testing procedures
- Quality improvements

## Development Workflow

1. **Fork the Repository**
   ```bash
   git clone https://github.com/YourUsername/LAN9252-EtherCAT-SPI-EVB.git
   cd LAN9252-EtherCAT-SPI-EVB
   ```

2. **Create a Feature Branch**
   ```bash
   git checkout -b improvement/your-improvement-name
   ```

3. **Make Your Changes**
   - Follow existing file structure and naming conventions
   - Update relevant documentation
   - Include clear commit messages

4. **Commit Your Work**
   ```bash
   git commit -m "Add: descriptive message of your changes"
   ```

5. **Push to Your Fork**
   ```bash
   git push origin improvement/your-improvement-name
   ```

6. **Submit a Pull Request**
   - Describe your changes clearly
   - Reference any related issues
   - Wait for review feedback

## Commit Message Guidelines

Use clear, descriptive commit messages:
- `Add: New feature description`
- `Fix: Bug fix description`
- `Update: Documentation or minor changes`
- `Refactor: Restructuring or optimization`
- `Remove: Deprecated or unused elements`

## Design Standards

### PCB Design
- Follow standard PCB design guidelines
- Maintain signal integrity best practices
- Ensure thermal management
- Consider manufacturability (DFM)
- Follow EtherCAT electrical specifications

### File Organization
- Keep Gerber files in dedicated folder
- Maintain BOM in CSV format
- Include documentation for any new layers
- Use consistent naming conventions

### Documentation
- Use Markdown format
- Include images/diagrams where helpful
- Keep instructions clear and beginner-friendly
- Update version history
